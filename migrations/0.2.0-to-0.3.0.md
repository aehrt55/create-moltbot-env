# Migration: 0.2.0 → 0.3.0

This migration renames the package from `@aehrt55` to `@marxbiotech`, adds deploy-time credential auto-loading, pre-deploy infrastructure checks (Cloudflare Queues + Access sync), and passes `CF_ACCESS_API_TOKEN` through CI/CD.

## Steps

### 1. Update package references

Replace all `@aehrt55/create-moltbot-env` references with `@marxbiotech/create-moltbot-env`:

```bash
# In .claude/commands/upgrade.md
sed -i '' 's/@aehrt55\/create-moltbot-env/@marxbiotech\/create-moltbot-env/g' .claude/commands/upgrade.md

# In README.md (footer line)
sed -i '' 's/@aehrt55\/create-moltbot-env/@marxbiotech\/create-moltbot-env/g' README.md
```

Also update `appRepo` in `.moltbot-env.json` if it still points to `aehrt55/moltbot-app`:
```bash
jq '.appRepo = "marxbiotech/moltbot-app"' .moltbot-env.json > tmp && mv tmp .moltbot-env.json
```

### 2. Add `scripts/ensure-queue.sh`

Create `scripts/ensure-queue.sh` by copying from a freshly scaffolded repo (`npx @marxbiotech/create-moltbot-env diff --json`), then make it executable:

```bash
chmod +x scripts/ensure-queue.sh
```

This script reads queue names from the overlay's `wrangler.jsonc` and creates any that don't exist via `wrangler queues create`. If no `queues` config exists, it injects a default pattern (`{worker}-telegram` + `{worker}-telegram-dlq`). Idempotent — safe to run on every deploy.

### 3. Update `scripts/deploy.sh`

Replace `scripts/deploy.sh` with the new version. Key changes:

**a) Unset `CF_API_TOKEN` locally** (after the CI check block):
```bash
else
  # Locally: unset deprecated CF_API_TOKEN so wrangler uses OAuth instead of a
  # potentially wrong token (e.g. CF Access token leaked into the env)
  unset CF_API_TOKEN 2>/dev/null || true
```

**b) Add credential auto-loading** (after the `APP_REPO` case block, before SOPS validation):
```bash
# --- Credential loading (local only) ---

load_credential() {
  local key="$1"
  local source=$(jq -r --arg k "$key" '.credentials[$k].source // empty' "$CONFIG")
  if [[ "$source" == "keychain" ]] && command -v security &>/dev/null; then
    local account=$(jq -r --arg k "$key" '.credentials[$k].keychainAccount' "$CONFIG")
    local service=$(jq -r --arg k "$key" '.credentials[$k].keychainService' "$CONFIG")
    security find-generic-password -a "$account" -s "$service" -w 2>/dev/null || true
  fi
}

# Load SOPS_AGE_KEY from keychain if not set
if [[ -z "${SOPS_AGE_KEY:-}" ]]; then
  SOPS_AGE_KEY=$(load_credential sopsAgeKey)
  export SOPS_AGE_KEY
fi

# Load CF_ACCESS_API_TOKEN from keychain if not set (for sync-access pre-deploy check)
if [[ -z "${CF_ACCESS_API_TOKEN:-}" ]]; then
  CF_ACCESS_API_TOKEN=$(load_credential cfAccessApiToken)
  export CF_ACCESS_API_TOKEN
fi
```

**c) Add pre-deploy checks** (after the `echo "Deploying..."` line, before cloning):
```bash
# 0. Pre-deploy: ensure infrastructure is in sync
echo "Pre-deploy checks..."
bash "$SCRIPT_DIR/ensure-queue.sh" "$ENV_NAME"

# sync-access needs CF_ACCESS_API_TOKEN (separate from CLOUDFLARE_API_TOKEN)
if [[ -n "${CF_ACCESS_API_TOKEN:-}" ]]; then
  bash "$SCRIPT_DIR/sync-access.sh" "$ENV_NAME"
else
  echo "  Skipping Access sync (CF_ACCESS_API_TOKEN not set)"
fi
```

### 4. Update `Makefile`

Add `ensure-queue` to the `.PHONY` line:
```makefile
.PHONY: deploy secret wrangler merged-config edit-secrets push-secrets sync-access ensure-queue
```

Add the target (before `edit-secrets`):
```makefile
# Ensure Cloudflare Queues exist for overlay config
ensure-queue:
	@bash $(ROOT)/scripts/ensure-queue.sh $(ENV_NAME)
```

### 5. Update `.github/workflows/deploy.yml`

Add `CF_ACCESS_API_TOKEN` to the deploy step's `env` block:
```yaml
        env:
          APP_REPO: ${{ steps.config.outputs.app_repo }}
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          CF_ACCESS_API_TOKEN: ${{ secrets.CF_ACCESS_API_TOKEN }}
          SOPS_AGE_KEY: ${{ secrets.SOPS_AGE_KEY }}
```

Then add `CF_ACCESS_API_TOKEN` as a repository secret in GitHub Settings > Secrets and variables > Actions.

### 6. Update `.moltbot-env.json` version

```bash
jq '.meta.version = "0.3.0" | .meta.lastUpgrade = "'"$(date +%Y-%m-%d)"'"' .moltbot-env.json > tmp && mv tmp .moltbot-env.json
```

### 7. Verify

1. Run `make -C overlays/<any-env> ensure-queue` to verify the script works
2. Run `make -C overlays/<any-env> deploy` to verify pre-deploy checks run correctly
3. Check scripts are executable: `ls -la scripts/ensure-queue.sh`
