# Migration: 0.2.0 → 0.3.0

This migration renames the package from `@aehrt55` to `@marxbiotech`, adds deploy-time credential auto-loading, pre-deploy infrastructure checks (Cloudflare Queues + Access sync), passes `CF_ACCESS_API_TOKEN` through CI/CD, and allows `version.txt` to accept any git ref (SHA, tag, or branch).

## Prerequisites

- **moltbot-app >= v2026.2.26-0** — This migration adds Cloudflare Queues infrastructure (ensure-queue.sh auto-injects queue config). The app must export a `queue()` handler, otherwise `wrangler deploy` will fail with "Queue handler is missing". Update your app's `version.txt` to `v2026.2.26-0` or later before applying this migration.

## Steps

### 1. Update package references

Replace all `@aehrt55/create-moltbot-env` references with `@marxbiotech/create-moltbot-env`:

```bash
# In .claude/commands/upgrade.md
sed 's/@aehrt55\/create-moltbot-env/@marxbiotech\/create-moltbot-env/g' .claude/commands/upgrade.md > tmp && mv tmp .claude/commands/upgrade.md

# In README.md (footer line)
sed 's/@aehrt55\/create-moltbot-env/@marxbiotech\/create-moltbot-env/g' README.md > tmp && mv tmp README.md
```

Also update `appRepo` (only if it still uses the old `aehrt55` slug) and version in `.moltbot-env.json`:
```bash
jq 'if .appRepo == "aehrt55/moltbot-app" then .appRepo = "marxbiotech/moltbot-app" else . end | .meta.version = "0.3.0" | .meta.lastUpgrade = "'"$(date +%Y-%m-%d)"'"' .moltbot-env.json > tmp && mv tmp .moltbot-env.json
```

### 2. Add `scripts/ensure-queue.sh`

Create `scripts/ensure-queue.sh` with the latest version from the scaffold templates, then make it executable:

```bash
chmod +x scripts/ensure-queue.sh
```

This script reads queue names from the overlay's `wrangler.jsonc` and creates any that don't exist via `wrangler queues create`. If no `queues` config exists, it injects a default pattern (`{worker}-telegram` + `{worker}-telegram-dlq`). Idempotent — safe to run on every deploy.

Key details:
- Only unsets `CF_API_TOKEN` when running locally (not in CI)
- Reads `cfAccountId` from `.moltbot-env.json` and exports as `CLOUDFLARE_ACCOUNT_ID` (required for multi-account wrangler)
- Uses create-and-check strategy: attempts `wrangler queues create`, treats "already taken" (error code 11009) as success
- Exits with error if queue injection fails (no silent fallthrough)

### 3. Replace `scripts/deploy.sh`

Replace `scripts/deploy.sh` entirely with the latest version from the scaffold templates, then make it executable:

```bash
chmod +x scripts/deploy.sh
```

Key changes from the previous version:

- **Unset `CF_API_TOKEN` locally** — prevents stale CF Access token from interfering with wrangler OAuth
- **Credential auto-loading** (local only, guarded by `CI` check) — `load_credential` function reads keychain credentials from `.moltbot-env.json` config, with warning messages on all failure paths (missing `security` command, bad config, keychain lookup failure, unknown source type). `load_credential` returns non-zero on retrieval failure; callers use `|| true` to allow graceful degradation while preserving the warning messages
- **Pre-deploy infrastructure checks** — runs `ensure-queue.sh` and conditionally `sync-access.sh` before cloning/deploying
- **Flexible version.txt** — removes SHA-only validation; `version.txt` now accepts any git ref (SHA, tag, or branch). Logs the resolved SHA after checkout for traceability
- **Secrets temp file cleanup** — updates the `trap` to include the secrets temp file path, ensuring decrypted secrets are cleaned up even if `wrangler secret bulk` fails

### 4. Update `Makefile`

Add `ensure-queue` to the `.PHONY` line:
```makefile
.PHONY: deploy secret wrangler merged-config edit-secrets push-secrets sync-access ensure-queue
```

Add the target (before `edit-secrets`):
```makefile
# Ensure Cloudflare Queues exist for overlay config
ensure-queue:
	@bash $(ROOT)/scripts/ensure-queue.sh $(ENV_NAME)
```

### 5. Update `.github/workflows/deploy.yml`

Add `CF_ACCESS_API_TOKEN` to the deploy step's `env` block:
```yaml
        env:
          APP_REPO: ${{ steps.config.outputs.app_repo }}
          CLOUDFLARE_API_TOKEN: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          CF_ACCESS_API_TOKEN: ${{ secrets.CF_ACCESS_API_TOKEN }}
          SOPS_AGE_KEY: ${{ secrets.SOPS_AGE_KEY }}
```

Then add `CF_ACCESS_API_TOKEN` as a repository secret in GitHub Settings > Secrets and variables > Actions.

### 6. Add `.claude/commands/ensure-queue.md`

Create `.claude/commands/ensure-queue.md` with the latest version from the scaffold templates. This Claude Code command provides an interactive workflow for ensuring Cloudflare Queues exist for an environment — it calls `scripts/ensure-queue.sh` under the hood and guides the user through environment selection and config verification.
